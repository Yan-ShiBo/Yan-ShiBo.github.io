
# 闫士博个人主页深度改版说明

## 这次改版做了什么

### 1. 统一设计系统
- 把原来多个页面各写一套内联样式，改成共享的 `resource/css/site.css`。
- 全站统一卡片、按钮、导航、页眉、页脚、间距、色彩与响应式规则。
- 增加暗色模式、移动端抽屉菜单、返回顶部、复制邮箱等交互。

### 2. 页面结构重构
保留原有核心页面，并补充了更适合展示技术能力的页面：
- `index.html`：新首页，突出研究身份与方法链
- `introduction.html`：个人档案与荣誉（完整保留内容并重新组织）
- `research.html`：**新增**
- `projects.html`：**新增**
- `resume_online.html`：在线简历摘要
- `resume.html`：PDF 简历页
- `stats.html`：访问统计页
- `404.html`：**新增**
- `home.html`：改为跳转到主页，避免重复入口

### 3. 双语结构升级
- 新增 `/en/` 英文目录，形成真正可分享的英文页面，而不是同页隐藏切换。
- 每个中文页面都有对应英文页：
  - `/index.html` ↔ `/en/index.html`
  - `/introduction.html` ↔ `/en/introduction.html`
  - `/research.html` ↔ `/en/research.html`
  - `/projects.html` ↔ `/en/projects.html`
  - `/resume_online.html` ↔ `/en/resume_online.html`
  - `/resume.html` ↔ `/en/resume.html`
  - `/stats.html` ↔ `/en/stats.html`

### 4. SEO / 分享 / 索引优化
- 增加每页唯一 `meta description`
- 增加 `canonical`
- 增加 `hreflang`
- 增加 `manifest.webmanifest`
- 增加 `robots.txt`
- 增加 `sitemap.xml`
- 首页与个人档案页加入 `JSON-LD` 结构化数据

### 5. 移动端与可访问性优化
- 原站点在小屏幕下会直接隐藏主导航；新版改为抽屉菜单。
- 所有主要按钮统一做到更易点击。
- 增加 skip link、语义化 header / nav / main / footer、aria-current、减少运动适配。

## 部署建议

1. 将本目录中的 HTML / CSS / JS 文件覆盖到你的 GitHub Pages 仓库。
2. 保留你原仓库已有的资源目录：
   - `Homepage_files/photo.jpg`
   - `Homepage_files/resume.png`
   - `Homepage_files/Shibo_Yan-CV.pdf`
   - `Homepage_files/index.ico`
   - `resource/font-awesome-4.7.0/...`
3. 提交并推送到 GitHub Pages。
4. 推送完成后检查：
   - 首页和英文首页是否都能打开
   - `research.html`、`projects.html` 是否可访问
   - 统计页是否能在未屏蔽脚本的环境下加载公开统计
   - 手机端抽屉菜单与暗色模式是否正常
